# Railway Production Deployment

## Environment Variables Required

Set these in Railway dashboard → Variables:

```bash
# Required
NODE_ENV=production
MONGODB_URI=your_mongodb_connection_string

# AI Provider
AI_PROVIDER=groq
GROQ_API_KEY=your_groq_api_key
GEMINI_API_KEY=your_gemini_api_key

# Auth
JWT_SECRET=your_secure_random_string

# Email
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_app_password
FROM_EMAIL=noreply@yourdomain.com

# Frontend
FRONTEND_URL=https://your-vercel-app.vercel.app

# Redis - Use Railway Redis addon or external Redis
# If using Railway Redis: Railway will auto-inject REDIS_URL
# If not: Leave empty, redis-memory-server will handle it (requires build tools)
```

## Deployment Steps

1. **Create Railway Project**
2. **Deploy from GitHub** → Select `piyushagarwal-55/Flowforge`
3. **Set Environment Variables** (copy from above)
4. **Railway auto-detects** Node.js and uses nixpacks.toml config
5. **Wait for build** (~2-3 minutes)
6. **Get URL** from Railway dashboard
7. **Update Vercel** `NEXT_PUBLIC_BACKEND_URL` with Railway URL

## Optional: Add Railway Redis

For better performance in production:
1. In Railway → **New** → **Database** → **Redis**
2. Railway auto-injects `REDIS_URL` to your service
3. Backend will use it instead of redis-memory-server
